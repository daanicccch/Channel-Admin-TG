# Templates

## Default event post
- Length: 250-900 chars.
- Use this as the main mode for this channel.
- Structure:
  1. Sharp title line with event, not with format label.
  2. Short context block.
  3. Price / offer / loss / ratio block as lines.
  4. One short ending line, question, or reaction.

Example:
```html
💎 <b>Редкий персик ушёл слишком дёшево</b>

Пользователь улучшил <b>Precious Peach</b> и выбил чёрный фон. Офферы прилетели сразу, но дальше началось странное.

<b>Офферы начали прилетать сразу:</b>
— 100 000 ⭐️ (~1 400 TON)

<b>Факт сделки:</b>
— 8 000 ⭐️ (~90 TON)

<b>Потеря:</b>
— 1 310 TON (~$1 755)

Как это вообще назвать: миссклик или подарок для перекупа?
```

## Model lineup with links
- Length: 180-500 chars.
- Use when the source is a compact collection of models, emoji packs, sticker packs, or upgrade assets.
- The links are mandatory content here.
- Keep 2-5 useful Telegram links directly in the body of the post.
- Prefer one short intro line and then a compact linked list.

Example:
```html
🎒 <b>Mood Pack: все доступные модели</b>

Собрали, что уже доступно по <b>Mood Pack</b>:
— <a href="https://t.me/GiftChangesModels/196">emoji</a>
— <a href="http://t.me/addemoji/MoodPack_emoji_by_GiftChangesHelper1Bot">emoji pack</a>
— <a href="http://t.me/addstickers/MoodPack_by_GiftChangesHelper1Bot">stickers</a>
— <a href="https://t.me/GiftChangesModels/196">models with %</a>

Какой вариант нравится больше?
```

## Upgrade / teaser post
- Length: 180-600 chars.
- Use when the whole point is a movement in emoji pack, lineup reorder, or visible prep for upgrades.
- If a model is shown for a known gift, describe it as a future model or upgrade model for that exact gift.
- Do not call it a `new variant` unless the source explicitly says a separate new gift is coming.
- Do not write that it will `appear in the game`; say `появится в Telegram`, `в подарках`, or `в апгрейдах`, if release is not confirmed yet.

Example:
```html
⏫ <b>Улучшения уже близко?</b>

В эмодзи-паке креативного директора 7 подарков подняли в начало набора. Обычно такие движения не происходят просто так.

🎁🎁🎁🎁🎁🎁🎁

За месяц он уже светил готовые модели именно для этих коллекций.
```

Alternative example:
```html
🍦 <b>У «Пломбира» засветилась новая модель</b>

Креативный директор показал одну из будущих моделей для <b>Пломбира</b>.
```

## Trade / exchange post
- Length: 250-800 chars.
- Structure:
  1. Exchange headline.
  2. What went in.
  3. What came out.
  4. Why the trade matters.

Example:
```html
🔁 <b>26 Heart Locket обменяли на 1 Plush Pepe</b>

В сделку ушли 26 <b>Heart Locket</b> плюс редкие подарки с чёрным фоном.

На выходе:
— 1 <b>Plush Pepe</b>

Если оценка входа верная, это сильный ориентир по цене редкого Pepe вне витрины.
```

## Service-backed market post
- Length: 250-700 chars.
- Use only when there is a real movement worth highlighting.
- Mention no more than 2-3 gifts in one post.
- Prices must go on separate lines if there are several names.

Example:
```html
📊 <b>В лидерах сегодня не только Pepe</b>

По GiftCharts вверх подтянулись:
— <b>Durov's Cap</b> — 696.99 TON
— <b>Heroic Helmet</b> — 227.7 TON

peek тоже показывает движение в тех же коллекциях. Пока это больше похоже на ротацию внимания, чем на общий разнос всего рынка.
```

## Weekly
- Length: 700-1200 chars.
- This is still a Telegram post, not an article.
- Use 2-4 strongest events of the week max.
- One block = one concrete case.
- No giant intro and no giant conclusion.
- No persuasion to buy, no “иксы”, no motivational language.
- If one event clearly dominates, it is better to build the weekly around that one event plus 1-2 short supporting notes.
- If a block has prices or offers, use a short lead-in plus compact bullets.
- If the same fact has two units, combine them in one line instead of splitting into separate naked lines.

Example:
```html
🎁 <b>Что сильнее всего двигало рынок подарков на этой неделе</b>

<b>1. Апгрейд-сигналы</b>
В эмодзи-паке креативного директора снова двигали старые коллекции вверх. Для рынка это привычный маркер: если подарок подсветили, по нему сразу просыпается интерес.

<b>2. Сделки по редким подаркам</b>
На неделе снова были кейсы, где редкие позиции уходили слишком дёшево или, наоборот, собирали жирные офферы. Лучше всего работали посты, где был не общий “рост рынка”, а конкретная сделка с понятной потерей или переплатой.

<b>3. Метрики сервисов</b>
GiftCharts и peek на этой неделе были полезны как подтверждение точечных движений, а не как основа для целого поста. Если сервис не привязан к конкретному подарку, лучше его вообще не тащить в текст.

Неделя была скорее про отдельные сильные кейсы, чем про общий памп всего рынка.
```
