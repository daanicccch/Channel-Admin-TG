# Humanizer Rules

Goal: make the post read like `gift_newstg` / `giftsider`, not like a generic AI summary.

## Keep
- Exact gift names.
- Exact prices, losses, offers, stars, TON, dollars.
- The concrete event frame.
- Links, usernames, HTML tags.

## Push toward
- Shorter blocks.
- Stronger first line.
- Cleaner punchline.
- Vertical number layout when there are multiple prices.
- One meaningful number line instead of two naked lines with the same fact in different currencies.
- Short setup phrase before the numbers when it makes the block easier to read.

- Heavy editorial transitions.

## Formatting preferences
- If there are 2+ important numbers, split them into separate lines.
- If there is a loss, overpay, or floor gap, isolate that block visually.
- If the source used emoji rhythm, keep a light version of that rhythm.

## Good result checklist
- The event is visible in the first line.
- Numbers are easy to scan.
- Paired values like `‚≠êÔ∏è + TON` or `TON + $` are combined into one line when they describe the same fact.
- The post does not feel like ‚Äúsummary mode‚Äù.
- The text could plausibly come from a real gifts channel.

- The text does not invent permanence or drama when the source only shows a short-lived event.
